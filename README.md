# Streaming Data Pipeline

A real-time data pipeline using streaming concepts for ingestion, processing, and near-real-time analytics.

## Goals
- Ingest streaming events
- Process and aggregate in near real-time
- Store outputs for analytics dashboards
- Provide reliable, replayable processing

## Tech Stack (planned)
- Kafka (or alternative)
- Python
- Spark Streaming (optional)
- Docker

## Architecture
**Producer → Kafka → Stream Processor → Storage → Analytics**

## Roadmap
- [ ] Local Kafka with Docker
- [ ] Producer emitting events
- [ ] Consumer processing + aggregations
- [ ] Persist results
- [ ] Monitoring and metrics
